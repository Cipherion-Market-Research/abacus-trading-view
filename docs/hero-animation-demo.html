Hero Number Animation — Exploration

Five animation techniques for the Abacus Predictions hero win-rate counter. Each card animates 85.0 to 96.3% using a different approach. Click "Replay" to re-trigger. Card 5 is the recommended production variant — scroll down to trigger its IntersectionObserver.

01 — CSS @property Counter
Pure CSS @property
%
Tech: @property --num registered as <number>, CSS-transitioned from 85 to 96.3. counter() displays the integer part; JS reads computed --num each frame for the decimal. Zero animation JS — pure CSS interpolation drives the motion.
Pro: Simplest, GPU-composited. Con: counter() truncates to integer — needs a JS shim for the fractional digit.
02 — Per-digit Odometer Flip
CSS transform Staggered
%
Tech: Each character is a vertical strip of 0-9 (or ".") in a clipped container. translateY() positions the visible digit. Spring easing cubic-bezier(0.22, 1.4, 0.36, 1) with ~80ms stagger per digit.
Pro: Classic odometer feel, very satisfying. Con: DOM-heavy if you have many counters.
03 — Spring-Physics Interpolation
rAF Overshoot
85.0%
Tech: requestAnimationFrame with spring physics — stiffness 120, damping ratio ~0.7, mass 1. The number overshoots to ~97.1 then settles to 96.3. ~20 lines of spring math. Updates text content each frame.
Pro: Physically plausible motion, the overshoot feels "alive." Con: JS-driven, not composited.
04 — View Transition API
View Transitions Cross-fade
85.0%
Tech: document.startViewTransition() morphs between old and new number. ::view-transition-old scales down + blurs out, ::view-transition-new scales up + blurs in. Falls back to direct swap if unsupported.
Pro: Browser-native morph, minimal code. Con: Chrome/Edge only (as of 2026), no per-digit control.

05 — Combined: Odometer + Spring + IntersectionObserver
Recommended Spring Odometer IO
%
Tech: Per-digit odometer strips, driven by spring-interpolated values via requestAnimationFrame. Animation triggers only when the element enters the viewport via IntersectionObserver(threshold: 0.5). A subtle blue glow pulse fires when the spring settles. Before visibility, displays "85.0".
Why this variant: Combines the satisfying mechanical feel of the odometer, the organic motion of spring physics, and the performance win of lazy triggering. The glow pulse provides a completion signal. Production-ready.
